Contrast Testing

To comply with WCAG 2.0 you need an accessible luminosity contrast ratio between text and background colors as specified in WCAG 2.0 - Section 1.4.3 "Contrast". This page helps you check which text/background color combinations meet that ratio. One way to do this is to
  1. Install aXe accessibility testing tools chrome extension: Accessibility Testing with aXe
  2. Open this page in the Chrome browser, open the developer tools, and select the "aXe" tab
  3. On 'aXe' tab press the 'Analyze' button
  4. Contrast warnings and issues will be displayed in different categories
  5. To the left of the warning text click the arrow to see the individual violations, clicking on each violation will highlight the dom with issues on the page.

Disabled Content

Note that WCAG 2.0 - Section 1.4.3 "Contrast" says that text or images of text that are part of an inactive user interface component have no contrast requirement. Therefore we don't include disabled text colors like $textColorDisabled in the panels on this page.
